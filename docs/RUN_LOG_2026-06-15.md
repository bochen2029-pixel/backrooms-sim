## What shipped, in order

### M27 — Procedural stairs → `m27-green` (commit `26eafe8`; + validator `2602675`)
The first true vertical connectivity: you can climb floor→floor in the streamed world.
- **`gen::stair_at(seed, level, cx, cz)`** — pure, total. Hybrid placement: a density scatter
  (`stair_hash % kStairDensityN=13`) **plus a 4×4-superblock backstop** (a Poisson-empty block still
  gets one canonical up-stair) → every superblock holds ≥1 up-stair, so the stack is connected within a
  bounded XZ distance (the Z-analogue of INV-3). Both floors read the SAME function — level L cuts its
  ceiling hole + builds the stairwell; L+1 reads `stair_at(…,L,…)` to cut its floor hole at the same cell
  (aligned by construction, INV-2 in Z, no neighbour query — the Z-analogue of `door_index`).
- **`GenerateChunk`** cuts the aligned holes + builds a **thin grounded riser-slab stairwell** (8 risers,
  inset, abutting — each thin-in-X so `ValidateChunkGeometry` accepts it as a "wall"; the first design,
  full-cell nested boxes, failed the validator). **`generate_layout`** carves the stair cell's interior
  walls open so it is always approachable; the pillar pass skips the stair cell.
- **Step-up locomotion** added to `core::move_and_collide` — a surgical pre-pass that mounts a low box
  (top within `kStepHeight=0.55 m` of the feet) when lifting onto it is clear. **Inert for full-height
  walls/pillars**, so all prior collision + determinism is bit-identical. (The axis-separated AABB
  resolver had no step-up, so stairs were unclimbable until this.)
- **`validate_vertical_connectivity`** — a Z flood-fill making "no floor is ever sealed" machine-checkable.
- **Proof:** `--ascend` climbs a real stair to **level 1** (bit-identical ×2), `--descend` still ↓ to -1,
  walk-bot 1 km × 0 stuck on all 5 seeds. **Goldens:** only the 2 ceiling-facing `m4/topdown` goldens
  changed (they look down at the new holes) — re-captured via `goldgen` + **ADR-053**; all 15 m5 shots +
  m1/m2/m7 byte-identical.

### M28 — Vertical streaming + see-through → `m28-green` (commit `a3c231e`)
Stand at a stairwell and see the next floor through the hole.
- **`StreamManager::update(center, extra_level)`** keeps a second concentric ring at one adjacent level,
  chosen by stance (climbing → `level+1`; else `level-1` to see down — the despair gradient). The 1-arg
  `update(center)` delegates to single-level, so run_game/soak/tests stay **bit-identical**.
- **See-through needed no renderer change** — chunk verts are baked at their `level_base_y` and the GPU
  upload cache is keyed by the full `ChunkKey`, so two levels at the same `(cx,cz)` draw at their true
  world-Y. Confirmed renderer-agnostic: the **DXR** `build_scene` builds a BLAS per resident chunk (no
  `(cx,cz)` dedup) → see-through works ray-traced too, zero extra code.
- **Proof:** `--vstream` — at a stairwell, residency `162 = 2×81` (bounded), both renders debug-clean, the
  floor above visibly drawn through the hole (`see_through_diff ~57–62`). **ADR-054.**

### M30 — Open shafts & the abyss → `m30-green` (commit `a17d7d4`; abyss `1a067bb`)
The second vertical system: a rare deep void you fall down — you sink faster than you climb.
- **`gen::shaft_at(seed, cx, cz)`** — pure, per-column, very rare (`hash % kShaftDensityN=1500`, the
  ~1.3 km cadence): a hashed cell + top level + depth (5–10). `shaft_passes(s, level)` lets any floor
  decide locally whether the void spans it (`[top-depth, top]`).
- **`GenerateChunk`** cuts the void's slice (floor open except the bottom landing, ceiling open except the
  top entry), additive to the M27 stair holes. **Soft-catch fall — no new physics:** the void has no
  floor, gravity drops the wanderer, the bottom level's solid floor catches it via the existing swept
  collision (a soft landing — there is no health/fail-state by design).
- **Abyss render:** `StreamManager::update(center, lo, hi)` streams a downward band; run_play/run_game open
  it over a shaft; you see several floors down the void, then black past the bounded ring = fog-to-black
  for free.
- **Proof:** `--shaftfall` (seeds 1/7/42 fall the full 10/9/9 floors and land, bounded, bit-identical ×2);
  `--abyss` (the depths show through the void, `abyss_diff ~21–60`, bounded `245 = 5×49`, debug-clean).
  **ADR-055.** Tracked polish (not blocking): the draft-audio telegraph + a deep-descent soak.

## Gotchas / lessons
- `ValidateChunkGeometry` is strict (grounded, thin-in-one-axis, non-overlapping) — the stairwell had to
  be thin riser-slabs, not full-cell boxes.
- The axis-separated AABB collision had **no step-up** — stairs needed new locomotion (kept inert for walls).
- PowerShell trap: `"...$seed: ..."` parses `$seed:` as a drive-qualified variable — use `${seed}:`.
- The `--vstream` camera first sat *inside* the staircase geometry (blocked view); moved to the clear
  low-approach strip.
- Surgical per-file commits can drop a sibling file (the validator) — end-of-turn `git status` caught it.
